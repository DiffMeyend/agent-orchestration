# Context Engine – Operator Template

Use this ritual whenever you need to rebuild context quickly (IT troubleshooting, family dynamics, creative work, etc.). Keep each pass lightweight; stop after you unlock the next good move.

---

## Setup
- Switch: start in Quiet unless the situation is already aligned; flip to Verbose only for teaching/explanation.
- Persona: default to Alchemist or Resonant to gather context, but log whichever persona actually runs the loop.
- Timebox: aim for one SCAN→FRAME→LEVERAGE pass in 5–10 minutes.

---

## SCAN – “What’s the scene?”
*(Orient, don’t solve yet. Pull from `docs/glossary.md#context-glossary-cheat-sheet` as needed.)*
- **Actors & Agents:** who/what is shaping this? note roles and leverage points.
- **Goals & Intentions:** why does this matter? what does each actor want/avoid?
- **Constraints & Rules:** policies, physics, bandwidth, emotional limits.
- **Environment / Temporal boundaries:** where + when is this playing out? urgency?
- **Meaning layers:** symbols/language, social/cultural signals, co-text cues.
- **Mental models:** what does each actor believe vs. know? list gaps or mismatches.
- ✅ Output: a minimum viable Composite Context Model (rough bullets are fine).

---

## FRAME – “What lens keeps this stable?”
- **Frames of reference:** which coordinates/explanations fit best? capture 1–2 candidate frames.
- **Reference points / anchors:** tag the baselines everyone can accept (time, scope, values).
- **Common ground:** list what is mutually understood; note any fragmentation/collapse risks.
- **Relevance filter:** decide what signals stay in scope for this pass.
- **Stabilization plan:** note how you’ll maintain/repair the frame if it drifts.

---

## LEVERAGE – “What move matters now?”
- **Desired outcome:** restate the goal in terms of the chosen frame.
- **Feasibility check:** re-run constraints/rules vs. available levers.
- **Environment (systems view):** what inputs or variables can you tweak?
- **Action selection:** pick one concrete step (operator, snippet, MCP tool) that advances the frame while respecting constraints.
- **Hand-off notes:** if another persona/operator will run the step, capture the intent_label and required context.

---

## Close / Next Loop
- Log any updates to context packets or history.
- If ambiguity remains, start another SCAN with the new information; otherwise execute the chosen action.
